# 🛑 서비스 종료 가이드

배민커넥트 라운지 웹사이트 종료 절차 안내입니다.

## 📋 종료 절차

### 1단계: 데이터 백업 (선택사항)

중요한 데이터가 있는 경우에만 백업하세요.

#### Supabase 데이터 백업 (선택사항)

```bash
# Supabase 대시보드에서 직접 백업
# 1. Supabase 대시보드 접속
# 2. Table Editor에서 각 테이블 선택
# 3. Export 기능 사용하여 CSV/JSON 다운로드

# 또는 스크립트 사용 (백업 스크립트 실행)
node scripts/backup-supabase.js
```

#### Google Sheets 데이터 백업 (선택사항)

Google Sheets는 이미 클라우드에 저장되어 있으므로 별도 백업 불필요하지만, 필요시:
- Google Sheets에서 "파일 > 다운로드" 메뉴 사용

### 2단계: 종료 페이지 배포 ✅

종료 페이지가 빌드되었습니다. 이제 배포하세요:

```bash
# ✅ 빌드 완료 (이미 실행됨)
# build 폴더에 최종 파일이 생성되었습니다.

# 배포 (사용 중인 플랫폼에 따라)
# - Vercel: vercel --prod 또는 Git push (자동 배포)
# - Netlify: netlify deploy --prod 또는 Git push (자동 배포)
# - GitHub Pages: build 폴더 내용을 gh-pages 브랜치에 push
# - 기타: build 폴더 내용을 호스팅 서버에 업로드
```

**배포 확인사항:**
- [ ] 종료 안내 페이지가 정상적으로 표시되는지 확인
- [ ] 모든 라우트가 종료 페이지로 리다이렉트되는지 확인
- [ ] 모바일에서도 정상적으로 표시되는지 확인

### 3단계: 자동화 스크립트 중지

#### GitHub Actions 중지
현재 프로젝트에는 GitHub Actions 워크플로우가 설정되어 있지 않습니다.
만약 다른 저장소나 환경에 설정되어 있다면:
1. GitHub 저장소 접속
2. Settings > Actions > General
3. "Disable Actions" 선택 또는
4. `.github/workflows/` 폴더의 워크플로우 파일 삭제/비활성화

#### Cron Job 중지 (서버에 설정된 경우)
```bash
crontab -e
# 관련 cron job 주석 처리 또는 삭제
```

### 4단계: 비용 최적화

#### Supabase 프로젝트 일시 중지
1. Supabase 대시보드 접속
2. Settings > General
3. 프로젝트 일시 중지 (Pause Project)
   - **주의**: 일시 중지 시 데이터는 7일간 보관되며, 이후 삭제됩니다.
   - 영구 보관이 필요하면 데이터 백업 후 프로젝트 삭제

#### 호스팅 비용 최소화
- 정적 페이지만 유지하므로 호스팅 비용은 최소화됨
- Vercel/Netlify 무료 플랜 사용 가능
- 필요시 도메인만 유지하고 호스팅은 최소 플랜으로 변경

### 5단계: 도메인 처리 (선택사항)

#### 도메인 리다이렉트 설정
다른 서비스로 리다이렉트하려면:
- DNS 설정에서 CNAME 또는 A 레코드 변경
- 또는 호스팅 플랫폼에서 리다이렉트 규칙 설정

#### 도메인 연장 중지
더 이상 사용하지 않는다면:
- 도메인 자동 갱신 해제
- 만료일 확인 후 필요시 수동 갱신
